## Bugs de infraestructura / servidor

### SBUG-1: YouTube API key incorrecta
**Impacto**: Item YouTube no puede entrenar
**Config actual**: La variable `GOOGLE_AI_API_KEY=AIzaSyCe...` es una key de Google AI Studio. Funciona para embeddings (Gemini) pero NO para YouTube Data API v3 (devuelve 403).
**OAuth disponible**: El sistema tiene OAuth con scope `youtube.readonly` conectado y funcionando (se usa para Drive, Sheets, Docs). El código NO aprovecha este OAuth para YouTube.
**Solución server-side**: Si se quisiera un fix sin cambio de código, se podría:
1. En Google Cloud Console del proyecto de la key `AIzaSyCe...`, habilitar "YouTube Data API v3"
2. O crear una key separada con YouTube Data API habilitado y configurarla
3. Pero la solución correcta es el fix de código (BUG-1 del informe de código) que use OAuth en vez de API key.

### SBUG-2: Módulo scheduled-tasks no arranca
**Log**:
```json
{"level":50,"name":"kernel:loader","module":"scheduled-tasks","err":{"message":"Invalid explicit day of month definition"},"msg":"Failed to activate module"}
```
**Impacto**: Bajo en lab (no hay tareas programadas críticas). Pero si se depende de este módulo en producción, necesita revisión.
**Causa**: Una expresión cron registrada en BullMQ tiene un día de mes inválido. Puede ser un job registrado en una sesión anterior que persiste en Redis.
**Fix rápido**: Limpiar los repeatables de BullMQ en Redis:
```bash
docker exec lab-redis redis-cli -a <password> KEYS "bull:scheduled*" | xargs docker exec -i lab-redis redis-cli -a <password> DEL
```
Luego reiniciar para que se re-registren con expresiones válidas.
