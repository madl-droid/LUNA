# Plan 2: Extracción Zero-LLM + Decay Temporal

## Objetivo
Eliminar TODAS las llamadas LLM internas del módulo de calificación. La extracción pasa a ser code-only: el agentic loop (que ya tiene el contexto completo) pasa datos estructurados como parámetros del tool, y el tool solo hace merge + scoring + transición. Eliminar también el nightly cold-lead scoring con LLM.

## Pre-requisitos
- **Plan 1 completado** — config simplificado a single-framework, tipos actualizados.

## Contexto para el ejecutor

### El problema actual
Cuando el agentic loop detecta info de calificación en un mensaje, llama al tool `extract_qualification` pasándole el texto del mensaje. Este tool hace OTRA llamada LLM (task `extract_qualification`, ruteada a modelo `main` = caro) para extraer datos estructurados del texto. Esto es redundante: el LLM del agentic loop YA procesó el mensaje y YA entendió la info — hacer otra llamada LLM para "re-leer" el mismo texto es desperdicio puro.

### La solución
Convertir `extract_qualification` en un tool que recibe datos ya extraídos como parámetros (JSON). El LLM del agentic loop es quien extrae los datos como parte de su razonamiento natural, y los pasa al tool como input estructurado. El tool solo hace: merge con datos existentes → recalcular score → transicionar estado si corresponde.

**Antes:**
```
Agentic LLM → call extract_qualification({ message_text: "texto" })
  → extract-tool hace OTRA llamada LLM → parse JSON → merge → score → update DB
```

**Después:**
```
Agentic LLM → call extract_qualification({ extracted: {...}, confidence: {...} })
  → extract-tool hace merge → score → update DB (CERO LLM)
```

**Nuevo `buildToolDescription()`:**
```typescript
function buildToolDescription(configStore: ConfigStore): string {
  const config = configStore.getConfig()
  const { criteria } = config

  if (criteria.length === 0) {
    return 'Registra datos de calificación de leads. No hay criterios configurados.'
  }

  const criteriaDesc = criteria
    .filter(c => !c.neverAskDirectly)
    .map(c => {
      let desc = `${c.key} (${c.type}): ${c.name.en}`
      if (c.type === 'enum' && c.options) {
        desc += ` [${c.options.join(', ')}]`
      }
      return desc
    })
    .join('; ')

  const neverAsk = criteria
    .filter(c => c.neverAskDirectly)
    .map(c => c.name.en)
    .join(', ')

  let desc = `Registra datos de calificación extraídos de la conversación. `
  desc += `Dispara cuando el contacto mencione información relevante para calificar. `
  desc += `Criterios: ${criteriaDesc}. `
  if (neverAsk) {
    desc += `Criterios de inferencia (nunca preguntar directo, solo registrar si el lead lo menciona): ${neverAsk}. `
  }
  desc += `Incluir solo datos claramente expresados con confianza >= 0.4. No adivinar.`

  return desc
}
```

**Nuevo `handleExtraction()` simplificado:**
```typescript
async function handleExtraction(
  input: Record<string, unknown>,
  ctx: ToolExecutionContext,
  configStore: ConfigStore,
  registry: Registry,
): Promise<{ success: boolean; data?: unknown; error?: string }> {
  const extracted = input['extracted'] as Record<string, unknown> | undefined
  const confidence = input['confidence'] as Record<string, number> | undefined
  const disqualifyReason = input['disqualify_reason'] as string | undefined
  const contactId = ctx.contactId

  if (!extracted || Object.keys(extracted).length === 0) {
    return { success: true, data: { extracted: false, reason: 'no_data_provided' } }
  }
  if (!contactId) {
    return { success: false, error: 'Missing contactId in execution context' }
  }

  const config = configStore.getConfig()
  const db = ctx.db

  // Validate keys against configured criteria
  const validKeys = new Set(config.criteria.map(c => c.key))
  const validExtracted: Record<string, unknown> = {}
  const validConfidence: Record<string, number> = {}

  for (const [key, value] of Object.entries(extracted)) {
    if (validKeys.has(key)) {
      validExtracted[key] = value
      validConfidence[key] = confidence?.[key] ?? 0.5
    } else {
      logger.debug({ key }, 'Ignoring extracted key not in criteria config')
    }
  }

  if (Object.keys(validExtracted).length === 0) {
    return { success: true, data: { extracted: false, reason: 'no_valid_keys' } }
  }

  // Transactional merge + score + transition
  const client = await db.connect()
  try {
    await client.query('BEGIN')

    const freshRow = await client.query(
      `SELECT qualification_data, lead_status FROM agent_contacts
       WHERE contact_id = $1 FOR UPDATE`,
      [contactId],
    )

    const freshData = (freshRow.rows[0]?.qualification_data as Record<string, unknown>) ?? {}
    const freshStatus = (freshRow.rows[0]?.lead_status as QualificationStatus) ?? 'new'

    // Merge
    const mergedData = mergeQualificationData(
      freshData,
      validExtracted,
      validConfidence,
      config.minConfidence ?? 0.4,
    )

    if (disqualifyReason) {
      mergedData['_disqualified'] = disqualifyReason
    }

    // Score
    const scoreResult = calculateScore(mergedData, config)
    const newStatus = resolveTransition(freshStatus, scoreResult.suggestedStatus)

    // Persist
    await client.query(
      `UPDATE agent_contacts
       SET qualification_data = $1,
           qualification_score = $2,
           lead_status = COALESCE($3, lead_status),
           updated_at = NOW()
       WHERE contact_id = $4`,
      [JSON.stringify(mergedData), scoreResult.totalScore, newStatus, contactId],
    )
    await client.query('COMMIT')

    // Emit status change hook
    if (newStatus) {
      logger.info(
        { contactId, from: freshStatus, to: newStatus, score: scoreResult.totalScore },
        'Lead qualification status changed',
      )
      await registry.runHook('contact:status_changed', {
        contactId,
        from: freshStatus,
        to: newStatus,
      })
    }

    return {
      success: true,
      data: {
        extracted: true,
        fields: Object.keys(validExtracted),
        score: scoreResult.totalScore,
        status: newStatus ?? freshStatus,
        disqualified: scoreResult.disqualified,
      },
    }
  } catch (txErr) {
    await client.query('ROLLBACK')
    logger.error({ err: txErr, contactId }, 'Extraction merge failed')
    return { success: false, error: 'Extraction merge failed: ' + String(txErr) }
  } finally {
    client.release()
  }
}
```

### 3. `src/engine/proactive/jobs/nightly-batch.ts` — Eliminar LLM scoring

**Reescribir `scoreColdLeads()`:**

Actualmente hace una llamada LLM por cada cold lead para "opinar" si vale la pena reactivar. Reemplazar con recalculación por código:

```typescript
async function scoreColdLeads(ctx: ProactiveJobContext): Promise<void> {
  const config = getNightlyConfig(ctx)
  if (!config.scoringEnabled) return

  // Import scoring engine
  const { calculateScore } = await import('../../modules/lead-scoring/scoring-engine.js')
  const configStore = ctx.registry.getOptional<ConfigStore>('lead-scoring:config')
  if (!configStore) return

  const qualConfig = configStore.getConfig()

  // Query cold leads in batches (cursor-based)
  let offset = 0
  const batchSize = config.scoringBatchSize
  let reactivated = 0
  let total = 0

  while (true) {
    const result = await ctx.db.query(
      `SELECT ac.contact_id,
              COALESCE(ac.qualification_data, '{}') AS qualification_data,
              ac.qualification_score
       FROM agent_contacts ac
       JOIN contacts c ON c.id = ac.contact_id
       WHERE c.contact_type = 'lead'
         AND ac.lead_status = 'cold'
       ORDER BY ac.updated_at DESC
       LIMIT $1 OFFSET $2`,
      [batchSize, offset],
    )

    if (result.rows.length === 0) break

    for (const row of result.rows) {
      const qualData = typeof row.qualification_data === 'string'
        ? JSON.parse(row.qualification_data) as Record<string, unknown>
        : (row.qualification_data as Record<string, unknown>) ?? {}

      const scoreResult = calculateScore(qualData, qualConfig)

      // Only update if score changed
      if (scoreResult.totalScore !== (row.qualification_score ?? 0)) {
        await ctx.db.query(
          `UPDATE agent_contacts SET qualification_score = $1, updated_at = NOW()
           WHERE contact_id = $2`,
          [scoreResult.totalScore, row.contact_id],
        )
      }

      // Reactivate if score above threshold
      if (scoreResult.totalScore >= config.scoringThreshold && scoreResult.suggestedStatus === 'qualifying') {
        await ctx.db.query(
          `UPDATE agent_contacts SET lead_status = 'qualifying', updated_at = NOW()
           WHERE contact_id = $1 AND lead_status = 'cold'`,
          [row.contact_id],
        )
        await ctx.registry.runHook('contact:status_changed', {
          contactId: row.contact_id,
          from: 'cold',
          to: 'qualifying',
        })
        reactivated++
      }
    }

    total += result.rows.length
    offset += batchSize

    // Safety: max 1000 leads per run
    if (total >= 1000) break
  }

  logger.info({ traceId: ctx.traceId, total, reactivated }, 'Cold lead scoring complete (code-based)')
}
```

**Nota:** Esta implementación también resuelve el problema de "cargar todo a memoria" usando cursor (LIMIT/OFFSET). El `taskPool` con concurrency ya no es necesario porque no hay llamadas LLM (el cálculo es <1ms por lead).

**Nueva función `calculateDecay()`:**
```typescript
function calculateDecay(extractedAtISO: string | undefined, windowDays: number): number {
  if (!extractedAtISO || windowDays <= 0) return 1 // sin timestamp o sin decay = full score

  const extractedAt = new Date(extractedAtISO)
  const now = new Date()
  const ageMs = now.getTime() - extractedAt.getTime()
  const ageDays = ageMs / (1000 * 60 * 60 * 24)

  if (ageDays <= 0) return 1
  if (ageDays >= windowDays) return 0.3 // floor: datos muy viejos contribuyen 30%

  // Linear decay de 1.0 a 0.3 en windowDays
  return 1 - (0.7 * ageDays / windowDays)
}
```
