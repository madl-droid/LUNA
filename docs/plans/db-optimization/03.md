### REGLAS para la escritura del SQL:

1. **TODA tabla usa `CREATE TABLE IF NOT EXISTS`** — idempotencia absoluta
2. **TODO indice usa `CREATE INDEX IF NOT EXISTS`** — idem
3. **Tipos corregidos** segun overview.md:
   - `session_archives.session_id` → UUID (no TEXT)
   - `session_summaries_v2.session_id` → UUID (no TEXT)
   - `session_memory_chunks.session_id` → UUID (no TEXT)
   - `hitl_tickets.requester_contact_id` → UUID (no VARCHAR)
   - `hitl_tickets.session_id` → UUID (no VARCHAR)
   - `calendar_follow_ups.contact_id` → UUID (no TEXT)
4. **FKs nuevas** segun overview.md (con ON DELETE apropiado)
5. **CHECK constraints** en todas las columnas de estado/tipo:
   - embedding_status, source_type, content_type, status, direction, speaker, etc.
   - Usar vocabulario FINAL: 'embedded' (no 'done')
6. **`created_at`** en TODAS las tablas que no lo tengan
7. **NO incluir tablas eliminadas** (agents, task_checkpoints, v1 tables, summary_chunks)
8. **NO incluir `contacts.company_id`** sin FK — mantener el FK a companies(id)
9. **Seeds con ON CONFLICT DO NOTHING** — safe para re-ejecucion
10. **Ordenar CREATE TABLE** por dependencia: tablas padre antes que hijas (ej: contacts antes de sessions)
