## Pasos de ejecucion

### Paso 1: Eliminar TODOS los archivos de migracion

```bash
rm src/migrations/*.sql
```

### Paso 2: Escribir `001_beta-schema.sql`

Crear `src/migrations/001_beta-schema.sql`. Para CADA tabla del inventario:
1. Leer el archivo fuente indicado para obtener columnas exactas
2. Aplicar los fixes de tipo (TEXT→UUID) marcados con **FIX**
3. Agregar FKs marcadas con **AGREGAR FK**
4. Agregar CHECK constraints en columnas de estado/tipo
5. Agregar `created_at TIMESTAMPTZ DEFAULT now()` si no tiene
6. Escribir CREATE TABLE IF NOT EXISTS con el estado FINAL (no hay ALTERs en un squash)

**Estructura del archivo:**

```sql
-- ============================================================
-- LUNA Beta — Canonical Schema
-- Single migration: creates ALL tables, indexes, constraints.
-- ============================================================

-- Prerequisites
CREATE EXTENSION IF NOT EXISTS vector;

-- Helper: updated_at trigger
CREATE OR REPLACE FUNCTION update_updated_at()
RETURNS TRIGGER AS $$
BEGIN NEW.updated_at = now(); RETURN NEW; END;
$$ LANGUAGE plpgsql;

-- SECTION 1: Kernel tables
-- SECTION 2: Users
-- SECTION 3: Core domain
-- SECTION 4: Memory v2
-- SECTION 5: Commitments
-- SECTION 6: Pipeline
-- SECTION 7: Campaigns
-- SECTION 8: LLM & tools
-- SECTION 9: Subagents
-- SECTION 10: Knowledge

-- SECTION 12: Voice

-- SECTION 14: HITL
-- SECTION 15: Cortex Trace
-- SECTION 16: Templates
-- SECTION 17: Medilink
-- SECTION 18: Seed data
-- SECTION 19: Triggers
```

### REGLAS para la escritura del SQL

1. **TODA tabla usa `CREATE TABLE IF NOT EXISTS`**
2. **TODO indice usa `CREATE INDEX IF NOT EXISTS`**
3. **Tipos corregidos** segun el inventario (marcados con **FIX tipo**)
4. **FKs nuevas** segun el inventario (marcadas con **AGREGAR FK**)
5. **CHECK constraints** en columnas de estado/tipo: embedding_status, source_type, content_type, status, direction, speaker, list_type, etc.
6. **Vocabulario de embedding_status:** 'pending', 'queued', 'processing', 'embedded', 'failed', 'pending_review'. **SIN 'done'**
7. **`created_at`** en TODAS las tablas
8. **NO incluir tablas eliminadas**
9. **Seeds con `ON CONFLICT DO NOTHING`**
10. **Ordenar por dependencia FK:** tablas padre antes que hijas
11. **NO incluir ALTER TABLE** — todo en estado final
12. **Cada tabla con comentario breve** de proposito
