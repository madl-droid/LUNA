
Si hay errores, corregirlos. TODOS los errores deben resolverse antes de continuar.

### Paso 2: Verificar que NO quede DDL en modulos

```bash
# Buscar CREATE TABLE fuera de migrations
grep -rn "CREATE TABLE" src/ --include="*.ts" | grep -v "node_modules" | grep -v "src/migrations/"

# Buscar ALTER TABLE fuera de migrations
grep -rn "ALTER TABLE" src/ --include="*.ts" | grep -v "node_modules" | grep -v "src/migrations/"

# Buscar CREATE INDEX fuera de migrations  
grep -rn "CREATE INDEX" src/ --include="*.ts" | grep -v "node_modules" | grep -v "src/migrations/"
```

**Resultado esperado:** CERO lineas de output para cada busqueda.
**Excepciones permitidas:** Comentarios que expliquen la eliminacion, o strings en test files.


### Paso 6: Verificar FKs y orden de CREATE TABLE

En `001_beta-schema.sql`, verificar que:
1. Tablas padre se crean ANTES que tablas hijas
2. Cada FK REFERENCES apunta a una tabla que ya fue creada
3. No hay dependencias circulares (si las hay, una tabla necesita ALTER TABLE despues de la creacion)

- Campaign default — verificar keyword y nombre
