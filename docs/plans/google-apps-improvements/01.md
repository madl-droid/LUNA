## Tarea 2: Paginación + output formateado (S1 + S4)

**Archivo:** `tools.ts` — tool `sheets-read` handler (líneas 386-406)

### Cambios en definición del tool

Agregar 2 parámetros opcionales a `properties`:
```typescript
offset: { type: 'number', description: 'Fila de inicio para paginación (0-based, default 0). Excluye la fila de header.' },
limit: { type: 'number', description: 'Máximo de filas de datos a retornar (default 100, max 500).' },
```

### Cambios en handler

Reescribir el handler completo con esta lógica:

1. **Parsear parámetros:**
   ```
   offset = Math.max(input.offset ?? 0, 0)
   limit = Math.min(Math.max(input.limit ?? 100, 1), 500)
   ```

2. **Auto-detect range** (de Tarea 1 — ya implementado arriba)

3. **Leer rango completo:**
   ```
   data = await sheets.readRange(spreadsheetId, range)
   values = data.values
   ```

4. **Guard hoja vacía:**
   ```
   Si values.length === 0:
     retornar { range, message: 'Hoja vacía — sin datos', totalRows: 0 }
   ```

5. **Separar header de datos:**
   ```
   header = values[0]     // fila 0 es el header
   dataRows = values.slice(1)  // filas 1+ son datos
   totalDataRows = dataRows.length
   ```

6. **Paginar:**
   ```
   pageRows = dataRows.slice(offset, offset + limit)
   hasMore = (offset + limit) < totalDataRows
   ```

7. **Formatear output tabular:**
   ```
   Rango: Sheet1!A1:D2501
   2500 filas de datos × 4 columnas
   Mostrando filas 1-100 de 2500

   SKU | Nombre | Precio | Stock
   ────┼────────┼────────┼──────
   ABC001 | Pantalla 65" | 2500 | 15
   ABC002 | Soporte móvil | 350 | 42
   ...

   (2400 filas más — usa offset=100 para ver las siguientes)
   ```

   Implementación:
   - Header: `header.join(' | ')`
   - Separador: generar `────` por cada columna, unir con `─┼─`
   - Datos: cada fila `row.join(' | ')`
   - Si `hasMore`: agregar indicador con siguiente offset

8. **Retornar objeto estructurado:**
   ```typescript
   return {
     success: true,
     data: {
       formatted,        // string tabular completo
       totalRows: totalDataRows,
       columns: header.length,
       header,            // array con nombres de columnas
       offset,
       limit,
       hasMore,
       nextOffset: hasMore ? offset + limit : null,
     },
   }
   ```

### Notas
- La Sheets API NO soporta paginación server-side en `values.get` — siempre retorna todo el rango. La paginación es client-side (slice).
- El header SIEMPRE se incluye en la respuesta (no cuenta como fila de datos).
- El campo `formatted` es lo que el agentic loop consume. Los campos numéricos son metadata para que el agente sepa si necesita paginar más.
- El límite de truncación del agentic loop es 8,000 chars (en `src/engine/agentic/agentic-loop.ts:442`). Con el formato tabular, ~100 filas caben en ~4-6K chars dependiendo del ancho de columnas. El default de 100 es seguro.

## Tarea 4: Restaurar validaciones post-append (S3)

### Paso 4a: Nuevos métodos en servicio

**Archivo:** `sheets-service.ts` — agregar 2 métodos a la clase `SheetsService`

**Método 1: `getRowValidations(spreadsheetId, sheetTitle, rowIndex)`**
```
1. Llamar spreadsheets.get con fields: 'sheets.data.rowData.values.dataValidation'
   y rango limitado a la fila específica: `${sheetTitle}!${rowIndex+1}:${rowIndex+1}`
   (Sheets API usa A1 notation, rowIndex es 0-based)
2. Extraer array de DataValidationRule de cada celda de esa fila
3. Retornar: Array<DataValidationRule | null> (null = sin validación en esa columna)
```

Nota: El tipo DataValidationRule de la API de Google es:
```typescript
interface DataValidationRule {
  condition: {
    type: string  // ONE_OF_LIST, ONE_OF_RANGE, BOOLEAN, etc.
    values?: Array<{ userEnteredValue?: string }>
  }
  showCustomUi?: boolean
  strict?: boolean
}
```

No crear tipo propio — usar `Record<string, unknown>` o el tipo de googleapis.

**Método 2: `applyValidations(spreadsheetId, sheetId, validations, startRow, numRows)`**
```
1. Construir array de requests para spreadsheets.batchUpdate
2. Para cada columna que tenga validación (!= null):
   - Crear SetDataValidationRequest:
     {
       setDataValidation: {
         range: {
           sheetId,
           startRowIndex: startRow,
           endRowIndex: startRow + numRows,
           startColumnIndex: colIndex,
           endColumnIndex: colIndex + 1,
         },
         rule: validations[colIndex]
       }
     }
3. Ejecutar spreadsheets.batchUpdate con todas las requests
```

### Paso 4b: Integrar en sheets-append handler

**Archivo:** `tools.ts` — handler de `sheets-append` (línea ~452)

Modificar el handler para que DESPUÉS del append, restaure validaciones:

```
1. ANTES del append:
   - Obtener info del spreadsheet: sheets.getSpreadsheet(spreadsheetId)
   - Parsear sheetTitle del range (ej: "Sheet1" de "Sheet1!A:D")
   - Encontrar el sheet correspondiente para obtener sheetId y rowCount
   - Obtener validaciones de la última fila con datos:
     sheets.getRowValidations(spreadsheetId, sheetTitle, lastDataRow)

2. EJECUTAR el append:
   result = await sheets.appendRows(...)

3. DESPUÉS del append (solo si hay validaciones):
   - Si alguna columna tenía validación:
     sheets.applyValidations(spreadsheetId, sheetId, validations, lastDataRow + 1, newRowsCount)
   - Esto es fire-and-forget (wrap en try/catch, log warning si falla, NO bloquear respuesta)
```

### Notas
- La restauración de validaciones es **best-effort** — si falla, el append ya se hizo correctamente. Solo loguear warning.
- Para parsear el sheetTitle del range: regex `/^'?([^'!]+)'?!/` o split por `!` y strip comillas.
- El `rowCount` de `SheetProperties.sheets[].rowCount` da el total de filas del grid, NO las filas con datos. Para la última fila con datos, restar 1 del número de filas retornadas por `readRange` al hacer el append original. Alternativamente, usar `result.updatedRows` del append para saber cuántas filas se insertaron.
- **Dropdown chips** (los pills de colores) son internamente `ONE_OF_LIST` validations — esta implementación los cubre automáticamente.

## Tarea 6: Batch Edit para Sheets (NUEVO)

### Paso 6a: Tipo nuevo

**Archivo:** `types.ts` — después de `SheetProperties` (línea ~110)

```typescript
export interface SheetBatchOperation {
  type: 'write' | 'append' | 'clear' | 'find_replace'
  range?: string           // requerido para write, append, clear
  values?: string[][]      // requerido para write, append
  find?: string            // requerido para find_replace
  replacement?: string     // requerido para find_replace
  matchCase?: boolean      // opcional para find_replace
}
```

### Paso 6b: Método en servicio

**Archivo:** `sheets-service.ts` — nuevo método en clase `SheetsService`

```typescript
async batchEdit(
  spreadsheetId: string,
  operations: SheetBatchOperation[],
): Promise<{ results: Array<{ type: string; detail: unknown }> }>
```

Lógica de agrupación por tipo de API call:

```
1. Separar operaciones por tipo

2. WRITES — spreadsheets.values.batchUpdate:
   - Agrupar todas las ops type='write'
   - Llamar values.batchUpdate con data: [{ range, values }] para cada write op
   - valueInputOption: 'USER_ENTERED'
   - Agregar { type: 'write', detail: { updatedCells } } al resultado

3. APPENDS — ejecutar secuencialmente (cada append puede ser a distinto rango):
   - Para cada op type='append': llamar this.appendRows(spreadsheetId, range, values)
   - Agregar { type: 'append', detail: { updatedRows } } al resultado

4. CLEARS — spreadsheets.values.batchClear:
   - Agrupar todas las ops type='clear'
   - Llamar values.batchClear con ranges: [op.range, ...]
   - Agregar { type: 'clear', detail: { clearedRanges: N } } al resultado

5. FIND_REPLACE — spreadsheets.batchUpdate:
   - Agrupar todas las ops type='find_replace'
   - Construir array de FindReplaceRequest
   - Llamar spreadsheets.batchUpdate
   - Agregar { type: 'find_replace', detail: { occurrencesChanged } } al resultado

6. Retornar { results: [...] }
```
