# Plan 1: Fundación del Módulo Templates

## Objetivo
Crear el módulo `templates` completo con infraestructura: DB, Drive extensions, repository, service base, console UI para gestión de plantillas por el admin.

## Prerequisitos
- Ninguno (es el primer plan)

## Contexto arquitectónico
- Módulo tipo `feature` que depende de `google-apps` y `tools`
- Patrón: similar a `freight` (módulo wrapper que consume servicios de otro módulo)
- Las plantillas son archivos de Google Drive (Docs, Slides, Sheets) con placeholders `{KEY_NAME}`
- El admin registra plantillas pegando URL de Drive → el módulo auto-detecta keys → admin agrega descripciones

### 1.1 Agregar método `copyFile()` a `src/modules/google-apps/drive-service.ts`

```typescript
async copyFile(
  fileId: string,
  name: string,
  parentId?: string,
): Promise<DriveFile> {
  const requestBody: Record<string, unknown> = { name }
  if (parentId) requestBody.parents = [parentId]

  const res = await googleApiCall(
    (signal) => this.drive.files.copy({
      fileId,
      requestBody,
      fields: 'id, name, mimeType, size, createdTime, webViewLink, webContentLink, parents',
      supportsAllDrives: true,
    }),
    this.apiConfig,
    'drive.files.copy',
  )

  return {
    id: res.data.id ?? '',
    name: res.data.name ?? '',
    mimeType: res.data.mimeType ?? '',
    size: res.data.size ?? undefined,
    createdTime: res.data.createdTime ?? undefined,
    webViewLink: res.data.webViewLink ?? undefined,
    webContentLink: res.data.webContentLink ?? undefined,
    parents: res.data.parents ?? undefined,
  }
}
```

**Nota**: Usar `googleApiCall` wrapper como `downloadFile` ya lo hace (timeout + retry). El parámetro `signal` puede ignorarse internamente si `this.drive.files.copy` no lo soporta (la llamada igual será cancelada por timeout del wrapper).

```typescript
async shareFileAnyone(
  fileId: string,
  role: 'reader' | 'writer' | 'commenter' = 'reader',
): Promise<DrivePermission> {
  const res = await this.drive.permissions.create({
    fileId,
    requestBody: {
      type: 'anyone',
      role,
    },
    fields: 'id, type, role',
  })

  return {
    id: res.data.id ?? '',
    type: (res.data.type ?? 'anyone') as DrivePermission['type'],
    role: (res.data.role ?? 'reader') as DrivePermission['role'],
  }
}
```

**Nota**: `DrivePermission` type ya soporta `type: 'anyone'` (ver `src/modules/google-apps/types.ts` línea 74).

## Paso 2: Migración SQL

### 2.1 Crear `src/migrations/048_templates-v1.sql`

```sql
-- Migration 048: Document Templates v1
-- Plantillas de documentos y documentos generados desde plantillas.

-- Tabla de plantillas registradas por el admin
CREATE TABLE IF NOT EXISTS doc_templates (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  name TEXT NOT NULL,
  description TEXT DEFAULT '',
  doc_type TEXT NOT NULL CHECK (doc_type IN ('comparativo', 'cotizacion', 'presentacion', 'otro')),
  drive_file_id TEXT NOT NULL,
  mime_type TEXT NOT NULL CHECK (mime_type IN ('document', 'spreadsheet', 'presentation')),
  keys JSONB NOT NULL DEFAULT '[]',
  folder_pattern TEXT DEFAULT '',
  sharing_mode TEXT NOT NULL DEFAULT 'anyone_with_link'
    CHECK (sharing_mode IN ('anyone_with_link', 'requester_only')),
  enabled BOOLEAN NOT NULL DEFAULT true,
  created_at TIMESTAMPTZ DEFAULT now(),
  updated_at TIMESTAMPTZ DEFAULT now()
);

-- Tabla de documentos generados desde plantillas
CREATE TABLE IF NOT EXISTS doc_generated (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  template_id UUID NOT NULL REFERENCES doc_templates(id),
  contact_id TEXT,
  requester_sender_id TEXT,
  requester_channel TEXT,
  drive_file_id TEXT NOT NULL,
  drive_folder_id TEXT,
  web_view_link TEXT NOT NULL,
  doc_name TEXT NOT NULL,
  key_values JSONB NOT NULL DEFAULT '{}',
  doc_type TEXT NOT NULL,
  status TEXT NOT NULL DEFAULT 'created'
    CHECK (status IN ('created', 'shared', 'archived')),
  tags JSONB DEFAULT '{}',
  version INT NOT NULL DEFAULT 1,
  created_at TIMESTAMPTZ DEFAULT now(),
  updated_at TIMESTAMPTZ DEFAULT now()
);

-- Índices para búsqueda eficiente
CREATE INDEX IF NOT EXISTS idx_doc_generated_template ON doc_generated(template_id);
CREATE INDEX IF NOT EXISTS idx_doc_generated_contact ON doc_generated(contact_id);
CREATE INDEX IF NOT EXISTS idx_doc_generated_type ON doc_generated(doc_type);
CREATE INDEX IF NOT EXISTS idx_doc_generated_tags ON doc_generated USING gin(tags);
CREATE INDEX IF NOT EXISTS idx_doc_templates_type ON doc_templates(doc_type);
CREATE INDEX IF NOT EXISTS idx_doc_templates_enabled ON doc_templates(enabled) WHERE enabled = true;
```

**Notas sobre el schema:**
- `keys` es JSONB array: `[{key: "COMPANY_NAME", description: "Nombre de la empresa cliente"}]`
- `key_values` es JSONB object: `{"COMPANY_NAME": "ACME Corp", "PRICE": "$5,000"}`
- `tags` es JSONB object para búsqueda flexible: `{"brand": "Nike", "competitor": "Adidas", "company": "ACME"}`
- `requester_sender_id` + `requester_channel` identifica quién pidió el documento (para restricción de sharing)
- `version` se incrementa en cada re-edición
- GIN index en `tags` permite queries como `tags @> '{"brand": "Nike"}'`

## Paso 5: Service

### 5.1 Crear `src/modules/templates/service.ts`

Clase `TemplatesService` que encapsula lógica de negocio:

```typescript
export class TemplatesService {
  constructor(
    private db: Pool,
    private registry: Registry,
    private config: TemplatesConfig,
  ) {}

  // ─── Template CRUD ───
  async listTemplates(filters?): Promise<DocTemplate[]>
  async getTemplate(id): Promise<DocTemplate | null>
  async createTemplate(input: CreateTemplateInput): Promise<DocTemplate>
  async updateTemplate(id, input: UpdateTemplateInput): Promise<DocTemplate | null>
  async deleteTemplate(id): Promise<boolean>

  // ─── Key auto-detection ───
  async scanKeysFromDrive(driveFileId: string): Promise<TemplateKey[]>
  // Lee el documento de Drive, extrae texto, busca patrón {KEY_NAME}
  // Retorna lista de keys encontradas (sin descripción — admin la agrega después)

  // ─── Helpers ───
  getTemplatesByType(docType: DocType): Promise<DocTemplate[]>

  // ─── Generated doc queries ───
  async searchGeneratedDocs(query): Promise<DocGenerated[]>
  async getGeneratedDoc(id): Promise<DocGenerated | null>
}
```

**Lógica de `scanKeysFromDrive`:**
1. Obtener Drive service: `registry.getOptional<DriveService>('google:drive')`
2. Obtener Docs/Slides/Sheets service según el mimeType del archivo
3. Leer contenido del documento:
   - Document: `docsService.getDocument(fileId)` → extraer texto
   - Presentation: `slidesService.getSlideText(fileId)` → extraer texto (incluye speaker notes)
   - Spreadsheet: `sheetsService.readRange(fileId, range)` → concatenar celdas
4. Regex: `/\{([A-Z][A-Z0-9_]*)\}/g` → extraer keys únicas
5. Retornar: `[{ key: "COMPANY_NAME", description: "" }, ...]`

**Detección de MIME type desde Drive file ID:**
1. Llamar `drive.getFile(fileId)` para obtener metadata
2. Mapear mimeType de Google:
   - `application/vnd.google-apps.document` → `'document'`
   - `application/vnd.google-apps.spreadsheet` → `'spreadsheet'`
   - `application/vnd.google-apps.presentation` → `'presentation'`

### 6.2 API Routes del manifest

Montar bajo `/console/api/templates/`:

| Método | Path | Descripción |
|--------|------|-------------|
| GET | `list` | Lista todas las plantillas |
| GET | `get?id=UUID` | Obtiene una plantilla por ID |
| POST | `create` | Crea una nueva plantilla |
| PUT | `update` | Actualiza una plantilla existente |
| DELETE | `delete?id=UUID` | Elimina una plantilla |
| POST | `scan-keys` | Escanea keys de un archivo Drive: `{ driveFileId }` → `TemplateKey[]` |
| GET | `generated?docType=X&contactId=Y` | Lista documentos generados (con filtros) |
| GET | `generated-detail?id=UUID` | Detalle de un documento generado |

**Flujo de creación de plantilla en la consola:**
1. Admin pega URL de Drive en un campo
2. Frontend extrae file ID del URL (regex client-side)
3. Frontend llama `POST /scan-keys { driveFileId }` → recibe lista de keys + mimeType detectado
4. Admin ve keys en una tabla, agrega descripción a cada una
5. Admin llena: nombre, tipo de documento, folder pattern, sharing mode
6. Frontend llama `POST /create { name, docType, driveFileId, mimeType, keys, folderPattern, sharingMode }`

**Handler de `POST /scan-keys`:**
1. `parseBody<{ driveFileId: string }>(req)`
2. Llamar `service.scanKeysFromDrive(driveFileId)` → retorna keys + mimeType
3. `jsonResponse(res, 200, { keys, mimeType })`
